### 堆 + 懒删除，但是不是O(1)，是O(logn)

这个解法看起来是对的，实际上错误的。

假设当前堆中插入了3个值：$x_1, x_2, x_3$，对应的优先级分别是：$r_1, r_2, r_3$，优先级将$[0, 1]$分为3段，不妨假设$0 <= r_1 < r_2 < r_3 < 1$，那么第一次随机取出的就是$x_1$，并且概率也是$1/3$，但是再次把$x_1$放到堆中，就有问题了，假如第二次随机取出的是$x_2$，那么一定有$0 <= r_2 < r_1 < r_3 < 1$，即$[0, 1]$ 3段中第一段变大了，那么随着第一段越来越大，那么就会出现每次随机的都是同一个值，而一旦下一次随机换了个值，就导致第一段更大了... ...
